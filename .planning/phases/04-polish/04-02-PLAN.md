  <name>Task 1: Add --format flag and JSON output for messages, threads, search commands</name>
  <files>cmd/root.go, cmd/messages.go, cmd/threads.go, cmd/search.go</files>
  <action>
1. **cmd/root.go**: Add persistent flag `--format/-f` (string, default "text") to rootCmd. Add getter `GetOutputFormat() string`. Valid values: "text", "json". No validation at root level (commands handle it).

2. **cmd/messages.go**: Modify all RunE functions to check `GetOutputFormat()`:

   - `runMessagesList`: When format=json, collect results into a slice of structs `{ID, ThreadID, Snippet string}` and output with `json.MarshalIndent(results, "", "  ")` followed by `fmt.Println(string(jsonBytes))`. Text mode unchanged.

   - `runMessagesGet`: When format=json, build a struct with From, To, Subject, Date, Body, Snippet, Labels ([]string from msg.LabelIds), Attachments ([]attachmentInfo from task 04-01). Output as JSON. Text mode unchanged.

   - `runMessagesModify`: When format=json, output `{MessageID, LabelsAdded, LabelsRemoved []string}`. Text mode unchanged.

   - `messages get-attachment` (from 04-01): When format=json, output `{FilePath, Size int64}`. Text mode unchanged.

3. **cmd/threads.go**: Modify RunE functions:

   - `runThreadsList`: When format=json, collect `{ThreadID, Snippet, MessageCount int}` into slice, output as JSON.

   - `runThreadsGet`: When format=json, build struct with ThreadID, Messages (slice of {From, To, Subject, Date, Body string}). Output as JSON.

4. **cmd/search.go**: Modify `runSearch`:
   - When format=json, collect `{ID, ThreadID, Date, From, Subject, Snippet string}` into slice, output as JSON.

Use `encoding/json` from stdlib — no external dependency needed. Use a helper function `outputJSON(v interface{}) error` in a new file or inline that does MarshalIndent + Println, to avoid repeating the pattern. Place the helper in root.go or a small internal package — simplest is a function in root.go since all cmd files already import the cmd package.

Text output remains exactly as-is — JSON is additive only.
  </action>
